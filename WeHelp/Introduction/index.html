我畢業於資訊工程系，曾在公司的 MIS 單位工作，主要協助資料整理與系統支援。雖然並非軟體開發職務，但這段經驗讓我確定自己想走向軟體工程師的職涯，因此開始自學製作專案。

以前就追蹤彭彭老師的 YouTube 頻道，覺得講解內容很清楚易懂，也因此認識了 WeHelp 訓練營。過去的自學雖然讓我累積了一些基礎，但也發現獨自學習容易遇到盲點，缺乏系統化的訓練與團隊合作的經驗。

在資工系學習程式設計和軟體開發基礎，進入職場後持續自學網頁開發與專案實作，並在 GitHub 建立作品集與取得的認證，另外曾以 Django 製作專案，並透過 Docker 與 AWS 部署上線，還有 iT 邦幫忙撰寫技術文章，持續記錄學習心得與開發經驗，培養整理與分享的能力。

計劃每週投入約 63 小時（平均每天 9 小時），安排如下：

學習進度
  • 週一：接收任務，規劃當週學習與實作方向
  • 週二～五：上午刷題與技術學習；下午專案開發與除錯；晚上整理問題與筆記
  • 週六～日：專案收尾，補強面試相關知識
回報與檢討
  • 週三、週六：回報學習進度，整理並提出問題
  • 週日：專案進度檢討與成果回顧

  • 追蹤趨勢：關注國內外技術論壇與社群，掌握新興技術方向
  • 交流學習：參加講座與培訓，向業界人士了解實務應用
  • 實作產出：持續專案練習並撰寫技術文章，觀察外界回饋
  • 市場驗證：透過面試了解企業需求，檢視技術價值

在前公司配合其他單位進行資訊稽核時，主管因個人因素遲到，對方公司已經到場等候。我多次聯絡主管卻未獲回覆，為避免影響公司形象，只能先以主管名義接待，並依照既有流程提供已審核過的資料。主管到場後認為我「不清楚自己的位置」，當下讓我感到委屈與不滿。

雖然心裡有負面情緒，但我仍選擇保持冷靜，向主管平和地說明過程，並強調自己有即時回報。基於單位利益，我不希望爭論擴大，因此在當下以理性與公事公辦的態度處理。事後我檢討這段經歷，認為能做到不卑不亢、兼顧部門利益與人際關係，比單純表達情緒更重要。

這次經驗提醒我，面對壓力或誤解時，保持冷靜、專注於解決問題，才能把負面情緒轉化為正向的職場態度。

在開發這份網頁的過程中，我學到許多雖然基礎但十分關鍵的前端技術。由於這是一個靜態網頁，沒有涉及後端程式，因此我特別專注在版面設計與互動功能的實作，例如導覽列、響應式排版與摺疊內容。透過這些嘗試，不僅加深了我對前端基礎的理解，也在完成設計與功能時獲得了成就感。

技術重點
  • 前端技術 : HTML / CSS / JavaScript 並搭配 Bootstrap 5
  • 版面設計 : 一頁式響應式(RWD)，適配桌機與行動裝置
  • 互動功能 : 導覽列快速跳轉、Accordion 摺疊式 Q&A
  • 部署流程 : GitHub repo 管理，撰寫 README.md 和 透過 GitHub Pages 上線

我認為人工智慧的發展對軟體工程師而言既是挑戰，也是機會。一方面，人工智慧能處理例行與重複性工作，並在學習與知識整理上提供輔助，讓工程師能將更多心力投入在系統設計與創新，進一步提升效率與品質。另一方面，隨著人工智慧逐漸取代部分基礎工作，對工程師的要求也更高，必須具備良好的規劃能力與除錯能力，才能與人工智慧協作並發揮其價值。另外人工智慧衍生的新領域會帶來很多新的機會，所以我認為精進自己的技術和知識，並且保持對新技術的敏感度，是軟體工程師未來發展的重要方向。

我非常認同 WeHelp 的理念，也期盼能成為其中一員。我相信自己有足夠的學習動力與執行力，能夠跟上訓練的步伐並全力投入。希望能獲得這次機會，和大家一起在實戰中成長，往軟體工程師的目標邁進。
